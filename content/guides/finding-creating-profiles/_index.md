MobiFlight profiles are created by the community, for the community.

## Finding community profiles

For Microsoft Flight Simulator, profiles are shared
on [flightsim.to in the MobiFlight category](https://flightsim.to/miscellaneous/mobiflight-profiles). Additional profiles
are occasionally posted in the [MobiFlight Discord](https://mobiflight.com/discord). Use the Discord search feature to find
them.

For X-Plane profiles, profiles are shared in [their forum](https://forums.x-plane.org/index.php). Use the forum search feature
to find them.

To steps to use community profile, see the [using community profiles guide](/guides/using-community-profiles/).

> [!IMPORTANT]
> Profiles are specific to each aircraft. For example, a profile designed for the FlyByWire A320 will only work
> with that plane. It will not work A320 aircraft from other sim vendors.

## Creating custom profiles

If a profile doesn't exist for an aircraft, you can create a custom one and then share it back to the community. Start
by understanding the [basics of MobiFlight](https://docs.mobiflight.com/getting-started/), then build out the rest of the
profile.

For Microsoft Flight Simulator aircraft, the events under **Microsoft** / **Generic** typically work for basic functionality
like landing gear, parking brake, flaps, and auto-pilot functions. For advanced avionics like flight computers, use the
developer tools to uncover the events. Guides on how to do this are available for
[MSFS2020](https://www.badcasserole.com/uncovering-input-events-using-the-msfs2020-model-behavior-dialog/) and
[MSFS2024](https://docs.mobiflight.com/guides/input-events-2024/).

If you have trouble finding an event, start a thread in the appropriate simulator support channel in the
[MobiFlight Discord](https://mobiflight.com/discord). When posting, make sure to include the specific aircraft and any
events you have already tried.
